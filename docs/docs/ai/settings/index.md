# AI 设置

配置 AI 提供商、模型和交互模式，定制您的 AI 体验。

## 前置条件

在配置 AI 设置之前，请确保您已具备：

- 已安装 Chaterm（支持 AI 功能的版本）
- 使用内置模型，或拥有至少一个模型提供商的 API 密钥或凭证（支持的提供商请参见 [模型配置](/docs/ai/llms/)）
- 能够访问所选提供商的 API 端点的网络连接（或本地 Ollama 实例）

---

## 创建新的 AI 对话

每个 AI 对话都是一个独立的会话，拥有自己的模式、模型和上下文。您可以同时运行多个对话。

1. 点击左侧菜单中的**"AI"**打开 AI 对话面板。
2. 点击面板顶部的**"新建对话"**按钮。
3. 选择一种**交互模式**：
   - **Command** -- AI 为当前活跃终端生成命令
   - **Agent** -- AI 自主执行多步骤任务
4. 从下拉列表中选择一个**模型**。只有已配置的模型才会显示。
5. _（仅 Agent 模式）_ 在输入框中输入 `@` 选择一台或多台**目标主机**。
6. 输入您的消息并按 **Enter** 开始对话。

::: tip
您可以同时打开多个对话，每个对话使用不同的模式或模型。这对于比较模型响应或处理并行任务非常有用。
:::

各模式的详细说明请参见 [AI 对话](/docs/ai/dialogs/)。

## 配置模型提供商

要使用 AI 功能，您需要至少配置一个模型提供商。

1. 点击右上角齿轮图标打开**设置**。
2. 在左侧菜单中导航到**"模型"**选项卡。
3. 点击**"添加模型"**。
4. 从列表中选择一个 **API 提供商**：
   - **LiteLLM** -- 支持多种提供商的统一接口
   - **OpenAI** -- OpenAI 兼容 API
   - **Bedrock** -- Amazon Bedrock（企业级）
   - **Anthropic** -- Anthropic API
   - **DeepSeek** -- DeepSeek API
   - **Ollama** -- 本地部署的模型
5. 填写所选提供商的**认证信息**（API 密钥、端点 URL、区域等）。
6. 输入或选择**模型名称**。
7. 点击**"测试连接"**验证配置。
8. 测试成功后，点击**"保存"**。

::: warning
请妥善保管您的 API 密钥。如果怀疑密钥已泄露，请立即向提供商申请轮换密钥，并在 Chaterm 中更新配置。
:::

提供商的具体配置字段参考表和详细设置说明，请参见 [模型配置](/docs/ai/llms/)。

---

## 导出聊天记录

您可以导出任何对话用于分享或存档。

1. 打开您想要导出的对话。
2. 点击**导出**按钮（通常位于对话面板的右上角区域）。
3. 选择导出格式和保存位置。

导出的聊天记录会保留完整的对话内容，包括 AI 回复、命令和执行输出。
